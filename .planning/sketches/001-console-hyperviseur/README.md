# Sketch 001 : Console Hyperviseur

## Design Question
Le portfolio déguisé en console Proxmox VE : jusqu'où pousser la fidélité à
l'interface sans perdre un recruteur non-technique ? L'enveloppe thématique
doit servir le contenu (fiches projets, CV, métriques homelab), pas l'écraser.

## How to View
open .planning/sketches/001-console-hyperviseur/index.html

## Variants
- **A : Réplique fidèle** — chrome Proxmox dense et compact, mono partout, le clin d'œil maximal pour un public infra.
- **B : Console aérée** — même métaphore (arbre + console) mais typo plus grande, console "flottante" arrondie, respiration moderne.
- **C : Landing hybride** — hero classique lisible par n'importe qui (nom, titre, CTA CV) avec la console embarquée comme pièce centrale en dessous.

## What to Look For
- L'arbre de gauche (Datacenter → nodes → LXC) est-il immédiatement compréhensible ?
- La fiche projet (Problème → Stack → Résultat → Leçons) reste-t-elle lisible dans chaque enveloppe ?
- Le Summary du Datacenter comme CV : naturel ou forcé ?
- Variante C : le hero dilue-t-il le concept ou le rend-il accessible ?
- Les métriques (gauges animées) : crédibles, à la bonne place ?
- Mobile (bouton 📱 de la toolbar) : l'arbre passe en haut — acceptable ?
